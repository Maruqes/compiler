A Tiny Systems Language That Writes Its Own ELF

BixoLang is a handcrafted experimental language that emits raw ELF64 binaries, manages its own memory (brk + mmap), spawns Linux threads with clone(), synchronizes with futex joins, does direct syscalls, runs a socket server, and builds higher-level constructs from first principles.

See Code Explore Features

No libc • No linker • Just syscalls

Core Capabilities

Raw ELF Emission

Generates executable ELF64 with correct program headers, aligned segments, globals + string pool, and manual segment size patching.

Memory Primitives

Custom brk allocator layering small chunks + direct anonymous mmap for larger or page-aligned regions.

Threading

Spawns threads via clone(), manual stack alignment, parameter passing, and futex-based joins (CLONE_CHILD_CLEARTID semantics).

Networking

TCP sockets with socket / bind / listen / accept / send / read / close. Minimal HTTP responder built from syscalls only.

Parser & Backend

Custom recursive-descent parser in Go driving a SWIG-exposed C backend that emits machine code + data sections.

Experiment Platform

Sandbox for exploring calling conventions, stack discipline, alignment, and direct kernel interfaces.

Sample: Spawning Threads

// allocate 8 pages for stack\ndq stack = mmap(8);\naddThreadVariable(stack, 8*4096, 0x1234, 0, 2);\naddThreadVariable(stack, 8*4096, 0x5678, 1, 2);\ncreateThreadIds(&worker, stack, 8*4096, &tid, 0, 2);\nthreadJoin(&tid);

Threads created with manual stacks. Arguments staged at top, 16‑byte aligned so callee sees valid call frame.

Join uses futex wait on child TID cleared by kernel.

No libc, no pthreads—just syscalls.

Runtime Architecture

Memory

brk() chained blocks + mmap for arenas. Global/string alignment enforced.

Codegen

C backend emits code + ELF header, patches sizes post data emission.

Concurrency

Stack isolation, param marshalling, futex sync primitives.
